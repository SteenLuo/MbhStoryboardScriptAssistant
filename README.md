# 漫剧工坊

> AI 漫剧剧本与分镜生产工作台

这是一个本地 AI 工作流项目，用来把小说输入转成适合 **AI漫剧** 的剧本，再把剧本转成适合 AI 画面和视频生成的分镜。

项目范围刻意保持很窄：

1. 输入小说，生成 AI 漫剧剧本。
2. 基于剧本，生成 AI 漫剧分镜。

长期目标是搭建一套“技能”驱动的 AI 漫剧改编系统：它能针对不同类型、不同题材的小说选择合适的剧本和分镜规则，并通过反复使用沉淀经验，逐步完善自己的技能。

## 项目定位

这是一个可交付给他人使用的 **AI 漫剧创作工作台**，不只是一次性提示词集合。

对客户来说，它是一个 AI 漫剧创作工作台；对产品介绍来说，它是一个剧本到分镜的垂直 AI 生产系统；对技术团队来说，它是一个基于本地技能的 workflow harness；对营销包装来说，它也可以被称为 AI 漫剧创作智能体。

核心目标是：让使用者不断投喂小说、AI 漫剧剧本、AI 漫剧分镜和修改意见后，系统能逐步学会他的格式习惯、镜头偏好、题材规则和质量标准，做到越用越聪明。

## 本地网页入口

推荐启动：

```text
双击根目录的 启动助手.bat
```

关闭服务：

```text
双击根目录的 关闭服务.bat
```

维护人员也可以用命令行启动：

```powershell
node app/server.js
```

访问：

```text
http://127.0.0.1:17877
```

网页是对话式入口，支持多个对话窗口。右上角“设置”可配置模型供应商、Base URL、模型和 API Key；当前支持 DeepSeek 和 OpenAI（默认 `gpt-5.5`）。

发送区带基础验证：没有文字、也没有附件时不能发送；输入文字或添加附件后才会启用发送按钮，避免误发空消息。

顶部“工作台”是独立弹窗，可选择当前对话或历史运行记录，把输入、理解、剧本版本、剧本检查、分镜版本、分镜检查和归档学习渲染成流程思维导图。点击节点可查看内容，并可把被采纳的节点加入手动归档学习。

聊天气泡下方有“复制”按钮，可一键复制用户提示词或助手输出内容。

## 客户试用包

在根目录双击：

```text
打包客户试用版.bat
```

脚本会在 `dist/` 下生成两个同版本号的 zip：

- `MbhStoryboardScriptAssistant-CustomerTrial-Full-v版本号.zip`：完整初始包，包含初始 `skills/`。
- `MbhStoryboardScriptAssistant-CustomerTrial-NoSkillOverwrite-v版本号.zip`：更新包，不包含 `skills/`，用于发给已经有自有技能沉淀的客户，避免覆盖他的 `skills/`。

版本号会自动读取 `dist/` 中已有包名，取最高版本并把最后一位加一。例如已有 `v0.1.3`，下次会生成 `v0.1.4`。

打包脚本会排除本机私有和运行数据，包括 `.env`、`.env.local`、`app/config/deepseek.local.json`、`app/config/app.local.json`、`app/data/conversations/`、`app/runtime/`、`runs/`、既有学习记录和已有样例源文件。

客户包内包含 [使用说明.md](使用说明.md)，用于指导客户解压、启动、配置模型供应商 API Key、关闭服务和理解 `Full` / `NoSkillOverwrite` 两种包。

## 核心流程

```mermaid
flowchart LR
    A["小说输入"] --> B["输入整理"]
    B --> C["类型和题材判断"]
    C --> D["生成剧本"]
    D --> E["评审剧本"]
    E --> F["生成分镜"]
    F --> G["评审分镜"]
    G --> H["沉淀经验"]
    H --> C
```

## 三种使用场景

1. **小说到剧本再到分镜**：输入小说，先生成剧本；人工或系统调整剧本后，再生成分镜。
2. **已有剧本评审改写再到分镜**：提供剧本，先评审和改写，再生成分镜。
3. **剧本直接生成分镜**：提供已经认可的剧本，直接生成分镜。

以上三种场景默认都以 **AI漫剧生产** 为目标，不以传统影视拍摄或广播剧制作为默认目标。

## 工作原则

所有能力都必须服务于两个核心产物：AI 漫剧剧本和 AI 漫剧分镜。

如果一个能力不能让 AI 漫剧剧本或分镜更好、更稳定、更容易复用或更容易评审，就先不进入当前路线图。
