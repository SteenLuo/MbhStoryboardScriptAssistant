# M7 本地网页入口验收记录

日期：2026-06-11

## 一、目标

M7 的目标是把 AI 漫剧剧本分镜工作流包成可重复使用的本地入口。

根据当前要求，M7 不做复杂后台，而是做成类似 GPT、豆包、Gemini 的本地对话式网页：

- 一个设置页，用来配置 DeepSeek API。
- 一个对话框，用来自然输入小说、剧本、分镜、样例和反馈。
- 支持多个对话窗口同时存在。
- M5 学习闭环和 M6 技能路由不能省，只是不在界面上做复杂菜单。
- 支持客户试用交付，避免泄露本机 API Key、测试对话和运行记录。

## 四、网页功能归拢

当前网页入口已经具备以下能力：

- 类 GPT 的主对话框。
- 左侧多对话窗口。
- 对话重命名。
- 删除对话时仅从网页列表隐藏，不删除本地文件。
- 独立回收站弹窗，可恢复被隐藏的对话窗口。
- 上传文件或图片。
- 发送验证：空文本且无附件时不能发送，输入文字或添加附件后才启用发送按钮。
- 对话消息显示时间、token 用量、命中 skill 和意图标签。
- 右侧当前对话概览，用于快速定位长对话内容。
- 皮肤切换：浅色、深色、护眼。
- DeepSeek 设置弹窗：Base URL、模型、API Key 保存和连接测试。
- 学习状态页：M0-M6 完整性、学习闭环刷新。

## 五、发送验证和输入保护

发送验证放在网页前端执行，目标是避免客户误触空消息，并让发送按钮状态和真实提交逻辑一致。

当前规则：

- 输入框为空，且没有待发送附件时，发送按钮禁用。
- 输入框有文字，或已添加至少一个附件时，发送按钮启用。
- 输入内容变化时，会实时刷新发送按钮状态。
- 添加或移除附件后，会同步刷新发送按钮状态。
- 即使用户通过 Enter 触发表单提交，发送前仍会再次校验；不满足条件时直接返回，不创建空对话消息。
- 发送后会清空输入框和待发送附件，并把发送按钮恢复到不可发送状态。
- 发送按钮禁用时带有 `aria-disabled` 和提示文案，方便用户理解为什么不能发送。

相关实现：

- `app/public/app.js`：`canSendCurrentCompose`、`updateSendState`、`sendMessage`。
- `app/public/styles.css`：发送按钮禁用态样式。

## 六、DeepSeek 配置

配置方式：

- 网页设置页。
- `.env` 或 `.env.local`。
- `app/config/deepseek.local.json`。

默认配置：

```json
{
  "baseUrl": "https://api.deepseek.com",
  "model": "deepseek-v4-flash"
}
```

注意：

- `app/config/deepseek.local.json` 是本机私有配置，打包脚本会排除。
- 客户试用包中只保留 `app/config/deepseek.config.example.json`。
- 客户第一次使用时，需要在网页设置页填写自己的 DeepSeek API Key。

## 七、M5 / M6 保留方式

M6 技能路由保留在后端系统提示和本地 skill 读取中：

- 小说 -> 输入整理 -> 剧本生成 -> 剧本评审 -> 分镜生成。

- 每个对话保存为本地证据。
- 用户偏好、样例解释和质量反馈可在后续总结为学习记录。
- 质量下降时优先止损。
- 正式规则不因单次反馈静默修改。

## 八、启动和关闭

客户试用时，优先使用根目录脚本：

```text
启动助手.bat
关闭服务.bat
```

`启动助手.bat` 的行为：

- 检查当前电脑是否有 Node.js。
- 如果没有 Node.js，下载官方 LTS 安装包并打开安装器。
- 如果服务已启动，只打开浏览器页面。
- 如果服务未启动，弹出服务窗口运行 `node app/server.js`，等待服务可访问后打开页面。

`关闭服务.bat` 的行为：

- 按启动脚本记录的 PID 精准关闭本项目服务。
- 按端口区分服务，避免测试端口误关主服务。

维护人员也可以用命令行启动：

```powershell
node app/server.js
```

默认访问地址：

```text
http://127.0.0.1:17877
```

## 九、客户试用包

根目录打包入口：

```text
打包客户试用版.bat
```

实际打包逻辑：

```text
tools/New-CustomerTrialPackage.ps1
```

脚本会在 `dist/` 下生成两个同版本号的 zip：

- `MbhStoryboardScriptAssistant-CustomerTrial-Full-v版本号.zip`：完整初始包，包含初始 `skills/`。
- `MbhStoryboardScriptAssistant-CustomerTrial-NoSkillOverwrite-v版本号.zip`：更新包，不包含 `skills/`，用于发给已经有自有技能沉淀的客户，避免覆盖他的 `skills/`。

版本号规则：

- 扫描 `dist/` 中已有 zip。
- 取最高版本号。
- 每次运行把最后一位加一。
- 如需固定发布版本，可运行 `powershell -ExecutionPolicy Bypass -File .\tools\New-CustomerTrialPackage.ps1 -Mode Both -ReleaseVersion 0.9.0`。

网页下载入口：

- 左下角“下载应用”入口读取 `dist/` 中最新的 `Full` 和 `NoSkillOverwrite` 两个 zip。
- `Full` 显示为“包含技能”，适合首次安装或同步官方技能。
- `NoSkillOverwrite` 显示为“不包含技能”，适合更新客户应用且不覆盖客户已有技能。
- 页面只负责下载已生成安装包；未打包时必须提示先执行打包脚本，不能伪造下载链接。

打包排除项：

- `.env`
- `.env.local`
- `app/config/deepseek.local.json`
- `app/data/conversations/`
- `app/runtime/`
- `runs/`
- 既有学习记录和评测结果
- 既有样例源文件
- `.git` 和 `.gitignore`
