## 一、验收结论

M5 当前结论：**已完成，可验收。**

这里的“完成”不是指系统会自动深读所有材料、自动修改正式技能，而是指已经具备一个可追溯、可审查、可回退的自主学习闭环：

```text
样例 / 对话 / 反馈
-> 结构化记录
-> 候选规则或回归任务
-> 学习快照
-> 技能进化草案
-> 证据满足后再修改正式技能
```

## 五、学习止损机制

M5 已明确：

- 使用人员反馈质量下降时，优先级最高。
- 系统自检发现明显退步时，也可触发降质记录。
- 降质输出不能作为正向样例。
- 可疑候选规则先冻结或继续观察。
- 可疑正式规则不直接删除，先由技能进化草案建议收窄或回退。
- 重复降质必须进入 M5 回归评测任务。

## 六、正式规则保护

M5 已明确：

- 候选规则可以自动生成。
- 正式规则必须有证据来源。
- 单次对话反馈不能静默修改正式技能。
- 题材专属规则需要跨剧目证据或用户明确确认。
- 技能进化报告必须说明修改原因、证据和回退方式。

## 七、验收判断

M5 已满足：

- 使用越多，候选规则越丰富。
- 正式规则有证据来源。
- 技能改进可追溯、可审查、可回退。
- 不会因为一次错误反馈污染整个系统。
- 发现质量下降时，能进入记录和回归评测，而不是继续强化错误规则。
