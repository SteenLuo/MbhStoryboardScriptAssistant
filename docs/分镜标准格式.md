# 分镜标准格式

本文档记录当前项目的 AI 漫剧分镜标准格式。后续生成分镜时，应优先使用此格式。

本格式默认服务 AI 漫剧生产，重点不是传统拍摄现场调度，而是让每个镜头都能继续转成 AI 画面生成、AI 视频生成、字幕、音效和剪辑节拍。

## 一、场次格式

每个场次以场次信息开头：

```text
场次：1-1
地点：无
时间：无
人物：无
```

字段说明：

- `场次`：建议使用“集数-场次”格式，例如 `1-1`、`1-2`。
- `地点`：写明场景地点；如果没有明确地点，写 `无`。
- `时间`：写明时间和内外景，例如 `日 外`；如果没有明确时间，写 `无`。
- `人物`：写本场出现的人物；如果无人，写 `无`。

## 二、镜头格式

每个镜头按以下字段书写：

```text
镜号：1
景别：低角度侧面中景
运镜：缓推
情绪/动作：无垠星海之中，无数个由法则锁链缠绕的白色光团沉浮不定。
音效：星空BMG
台词：旁白VO：太初混沌，万界初开。
时长：3s
```

标准字段：

- `镜号`
- `景别`
- `运镜`
- `情绪/动作`
- `音效`
- `台词`
- `时长`

可选字段：

- `字幕`

## 三、字段顺序

推荐顺序：

```text
镜号：
景别：
运镜：
情绪/动作：
音效：
台词：
时长：
字幕：
```

如果原始材料中 `台词` 和 `音效` 顺序不同，也可以接受；但新生成分镜时建议统一为上面顺序。

## 四、写法要求

### 场次

同一个场次下可以包含多个镜头。进入新场次时，重新写：

```text
场次：
地点：
时间：
人物：
```

### 镜号

镜号应连续编号。跨场次后，镜号可以继续累加，不必从 1 重新开始。

### 景别

`景别` 字段必须同时承载景别、构图角度和拍摄视角，不再单独增加构图字段。常见写法包括：

- 远景
- 全景
- 中景
- 近景
- 特写
- 面部特写
- 俯拍特写
- 仰拍中景
- 低角度侧面中景
- 正三四仰拍近景
- 侧面俯拍中景
- 过肩中景
- 前景遮挡近景
- 全景-中景

景别应优先写成 `视角/构图 + 景别`，例如 `低角度侧面中景`、`正三四仰拍近景`、`侧面俯拍中景`。如果镜头是默认正面平视，可以省略构图和视角，只写 `中景`、`近景`、`特写`。

正面平视镜头占比必须控制在总镜数的 30% 到 40% 之间。只有基础景别且未注明构图/视角的镜头（如 `景别：中景`、`景别：近景`、`景别：特写`）默认按正面平视计入；明确写有侧面、正三四、过肩、前景遮挡、反打、仰拍、俯拍、低角度等信息的镜头不计入正面平视。

### 运镜

常见写法包括：

- 固定
- 缓推
- 快速推进
- 缓慢平移
- 下摇
- 下移
- 跟镜头
- 仰拍固定
- 俯拍固定
- 下移跟随
- 拉
- 摇
- 移
- 升格
- 降格
- 手持
- 甩镜头
- 晃镜头

如果运镜暂时没有明确设计，可以留空，但生成新分镜时应尽量补齐。

运镜原则：

- 不要全是固定镜头。
- 不要为了变化而乱运镜。
- 运动镜头占比必须控制在总镜数的 30% 到 40% 之间；`运镜` 字段不含 `固定` 的镜头按运动镜头计入。
- `运镜：固定`、`运镜：仰拍固定`、`运镜：俯拍固定` 等含 `固定` 的镜头按固定镜头计入。
- 禁止连续 3 个及以上运动镜头；连续两个运动镜头后，下一个镜头应使用固定类运镜。
- 推用于逐渐聚焦角色或物品。
- 拉用于逐渐展示角色周围环境。
- 摇用于角色第一人称视角观察四周环境或人物。
- 移用于机位从一个位置平移到另一个位置。
- 升格用于机位从低视角往高视角上升。
- 降格用于机位从高视角往低视角下降。
- 手持、甩镜头、晃镜头用于慌张、混乱、突发反应等情绪，但不要滥用。

### 构图

人物对话不要过多使用正面怼脸。可根据剧情使用正面、正四三、正反打、外反打、内反打、前景遮挡等构图。

单个人物说台词时，优先使用近景或中景，少用特写。不要频繁对同一角色做多角度、多景别切换。

需要切镜时，画面必须结合剧本当前情节、台词信息和人物关系重新设计，不强制切成某类反应或特写。如果使用反应镜头，必须根据当前台词含义、人物关系和场景状态写具体反应，不必总是看向正在说话的人；可以使用低头、停手、手指停住、肩背变化、话头停住、看向场景内已有物件或空间等低风险反应。不得连续套用 `眼神一顿，轻轻抿住嘴`、`A看向B，神情从迟疑转为专注`、`表情随台词变化`、`神情贴合当前台词` 等泛化模板。

画面视角不要全是平视。可根据内容使用平视、仰视、俯视、低视角、地面视角或特殊视角。

禁止连续使用相同景别、相同角度、相同构图拍摄同一画面内容。若相邻镜头的 `景别` 完全相同，或只差 `正面`、`平视`、`正面平视` 等默认省略词，且 `情绪/动作` 写了 `维持上一镜`、`继续上一镜`、`接上一镜`、`同一构图`、`同一机位`、`画面不变`，应更换景别、角度或构图，并结合剧本给出不同画面；如果没有新的画面信息，就合并同一画面。连续多个 `双人中景` 或 `正面平视双人中景` 拍同一段对坐对白属于违规。因长台词拆分产生的连续镜头也不得复制同一段画面动作，且不得同时保持相同景别/构图和相同运镜。

### 情绪/动作

这是分镜中最重要的字段，应写出具体画面，而不是抽象描述。

推荐写法：

- 写清楚画面主体。
- 写清楚人物动作。
- 写清楚表情和情绪。
- 写清楚关键道具或特效。
- 必要时用括号补充前景、背景、特效说明。

示例：

```text
情绪/动作：夏清站在人群前方，神情高傲，背景人群神情激动互相交谈。
```

### 台词

台词可以包含：

- 人物对白
- 旁白VO
- 系统音VO
- 画外音
- 人物内心 OS

示例：

```text
台词：旁白VO：太初混沌，万界初开。
台词：画外音：我不是刚氪了10个648好不容易出了一个金吗？
台词：凌天尊者OS：此女灵运极强，天赋惊人。
```

台词拆镜边界：

- 人物台词必须保真：禁止对剧本中的人物台词进行任何改动，不得改写、润色、同义替换、删减、扩写或新增人物说出口的话。
- 原文人物行里的动作提示可以移入 `情绪/动作`，但说出口的台词必须保留原文。
- 非空台词必须在台词内容前标注说话人或声音来源，格式为 `台词：说话人：原文台词`，例如 `台词：陈建军：秀娥，你今天真好看。`。不能写成 `台词：秀娥，你今天真好看。`。
- 旁白、画外音、OS、VO 也要标注来源，例如 `台词：旁白VO：我到这儿一年了。`、`台词：林秀娥OS：这个机会还没来。`；没有台词时可写 `台词：` 或 `台词：无`。
- 只有同一说话人的连续台词超过 20 字时，才因为台词长度拆成多个连续镜号。
- 同一说话人的连续台词总字数 ≤ 20 字时，禁止仅为了变化景别、角度、运镜或画面描述而拆成多个分镜。
- 同一说话人的多个短句如果中间没有他人反应、动作转折或关键信息层镜头，应合并到同一镜头。
- 拆分长台词时，不得把每个短句机械拆成单独镜号；应尽量把相邻短句合并到 20 字以内。
- 相邻两个镜号如果都是同一人物说话，且两句台词合并后仍 ≤20 字，应判定为拆得过碎，必须合并到同一镜号。例如 `林秀娥：您言重了。` 和 `林秀娥：您以前也是按规矩办事。` 不应拆成两个镜号。
- 拆分后的台词片段按顺序拼回去，必须与剧本原台词完全一致；如果拼不回原文，就是台词被改写。
- 长台词因超过 20 字拆成连续镜号时，不能只复制同一段 `情绪/动作` 再改台词；后续镜头必须让画面动作随台词片段推进，并更换景别/构图或运镜。
- 拆分长台词的后续镜头只保留两个要求：景别和画面内容不能完全一样；画面必须结合剧本当前情节和台词信息给出对应画面。除此之外，不限定后续镜头必须采用某类画面。没有新的画面信息时，不要把同一画面硬拆成重复镜头。

### 音效

音效可以写背景音乐、环境声、特效声。

示例：

```text
音效：星空BMG
音效：星空BMG＋破碎
音效：熙熙攘攘的人群
```

没有音效时可以留空：

```text
音效：
```

### 时长

统一使用秒数，例如：

```text
时长：3s
时长：4s
时长：6s
```

## 五、标准模板

```text
场次：
地点：
时间：
人物：

镜号：
景别：低角度侧面中景
运镜：
情绪/动作：
音效：
台词：
时长：
字幕：
```

## 六、生成分镜时的要求

后续由剧本生成分镜时，应遵守：

1. 先写场次信息，再写镜头。
2. 每个镜头必须有 `镜号`、`景别`、`运镜`、`情绪/动作`、`音效`、`台词`、`时长`。
3. `情绪/动作` 必须是具体画面。
4. `景别` 要写入必要的构图角度和拍摄视角，例如 `低角度侧面中景`、`正三四仰拍近景`；正面平视可只写 `中景`、`近景` 等基础景别。
5. 正面平视镜头占比控制在 30% 到 40% 之间。
6. 运动镜头占比控制在 30% 到 40% 之间，且禁止连续 3 个及以上运动镜头。
7. 非空 `台词` 必须标注说话人或声音来源，写成 `台词：说话人：原文台词`，并保留旁白、画外音、系统音等标记。
8. 人物台词必须保留剧本原文，只能按原文切段，不能改写、润色、删减、扩写或新增。
9. `时长` 使用 `3s`、`4s` 这类格式。
10. 如果需要字幕，放在镜头字段末尾。
11. 不使用表格作为默认输出格式，默认使用逐镜头文本块。
12. 默认按 AI 漫剧用途生成，画面主体、动作、情绪、字幕和音效要清楚。
13. 避免在单个镜头里堆叠过多主体、动作和复杂空间关系，降低后续 AI 生成难度。
