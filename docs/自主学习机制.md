# 自主学习机制

本文档说明本项目如何参考 `gstack` 和 `gbrain`，做出“越用越聪明”的能力。

## 一、目标

自主学习的目标不是让系统不受控制地自动改写自己，而是让系统在使用中逐步积累：

- 用户喜欢的 AI 漫剧剧本格式。
- 用户认可的 AI 漫剧分镜格式。
- 不同题材的改编规则。
- 每集爽点和高潮点安排习惯。
- 单集和整体剧情逻辑问题的常见修法。
- 常见错误和修正方式。
- 哪些规则经过验证，哪些还只是猜测。

最终效果是：同一个使用者用得越多，系统越懂他的 AI 漫剧剧本和分镜标准。

## 二、参考来源

### 从 `gstack` 借鉴

借鉴它的流程化能力：

- 先澄清需求。
- 再形成规格。
- 重要产物进入下一步前必须评审。
- 每一步都有明确输入、输出和验收标准。

落到本项目就是：

```text
输入整理 -> 剧本评审 / 改写 -> 分镜生成 -> 分镜评审 -> 学习记录
```

### 从 `gbrain` 借鉴

借鉴它的记忆和进化能力：

- 保存历史材料。
- 先检索再生成。
- 把经验沉淀成规则。
- 用评测样例验证规则是否有效。
- 技能文本本身可以被改进。

落到本项目就是：

```text
样例 / 反馈 / 运行记录
-> 候选规则
-> 评测验证
-> 人工确认
-> 正式技能或正式参考规则
```

## 三、学习分层

本项目采用五层学习结构。

### 第 1 层：原始材料层

位置：

```text
samples/
samples/_inbox/
```

内容：

- 小说原文。
- 剧本。
- 分镜。
- 草稿。
- 定稿。
- 用户随手丢进来的混合文件。

要求：

- 不要求整齐。
- 不直接当成正式规则。
- 先由 `sample-ingest` 分拣。

### 第 2 层：运行记录层

位置：

```text
runs/
```

内容：

- 本次输入。
- 本次输出。
- 使用了哪些技能。
- 用户怎么改。
- 评审发现了什么问题。

作用：

- 让系统知道每次生成发生了什么。
- 后续可以追溯某个规则从哪里来。

### 第 3 层：候选规则层

位置：

```text
learning/candidate-rules/
```

内容：

- 还没确认的规则。
- 从样例中提炼出的格式习惯。
- 从失败中总结的问题。
- 用户偏好的初步判断。

要求：

- 候选规则可以自动生成。
- 候选规则不能直接覆盖正式技能。
- 每条候选规则必须写证据来源。

### 第 4 层：评测验证层

位置：

```text
learning/evals/
```

内容：

- 类型判断评测样例。
- 剧本评审评测样例。
- 分镜生成评测样例。
- 技能修改前后对比记录。

作用：

- 判断某个规则是否真的让结果变好。
- 防止“听起来合理但实际变差”的修改。

### 第 5 层：正式规则层

位置：

```text
learning/accepted-rules/
skills/*/references/
```

内容：

- 已确认的格式规则。
- 已确认的用户偏好。
- 已确认的题材规则。
- 已确认的分镜标准。

要求：

- 必须能追溯证据。
- 必须说明适用范围。
- 必须能回退。

## 四、学习流程

```text
用户投喂材料
-> sample-ingest 分拣
-> 生成样例学习报告
-> 提炼候选规则
-> skill-evolution 评估是否值得升级
-> 使用评测样例验证
-> 用户或维护者确认
-> 写入正式规则
```

## 五、什么可以自动做

系统可以自动做：

- 扫描样例。
- 分拣材料。
- 提炼候选规则。
- 记录用户修改差异。
- 发现重复问题。
- 生成改进建议。
- 生成评测样例草稿。

## 六、什么不能自动做

系统不能静默做：

- 单次反馈直接改正式技能。
- 未确认样例直接变成正式规则。
- 删除历史样例。
- 覆盖用户原始材料。
- 为了一个题材破坏其他题材规则。

## 七、越用越聪明的具体表现

使用一段时间后，系统应该能做到：

- 知道用户默认喜欢哪种分镜格式。
- 生成分镜时自动使用项目标准格式。
- 识别某类剧本是否不适合直接进入分镜。
- 发现用户经常修改的地方，并提前规避。
- 针对甜宠、悬疑、仙侠、家庭伦理等题材使用不同策略。
- 在输出前参考过去的成功样例。
- 对技能修改给出证据，而不是凭感觉。

## 八、当前落地方式

当前已建立：

- `sample-ingest`：负责样例分拣和候选规则提炼。
- `skill-evolution`：负责把候选规则升级为可审查的技能改进建议。
- `samples/`：存放用户投喂材料。
- `samples/_reports/`：存放样例分拣报告。
- `docs/分镜标准格式.md`：已确认的分镜格式规则。
- `learning/candidate-rules/`
- `learning/accepted-rules/`
- `learning/evals/`
- `runs/`

下一步需要做的是：把真实运行结果和用户反馈写入 `runs/`，再由 `sample-ingest` 和 `skill-evolution` 逐步沉淀候选规则。
