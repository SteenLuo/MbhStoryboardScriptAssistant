# M8 流程思维导图和手动归档学习阶段记录

日期：2026-06-11

## 一、目标

M8 第一段不再把工作台做成第二个生成入口。

新的目标是：把一次创作流程渲染成可查看的树状思维导图，让客户能看清输入、理解、剧本版本、剧本检查、分镜版本、分镜检查和归档学习之间的关系。

主输入仍然只保留底部聊天框。工作台只负责查看流程、打开节点内容和整理人工采纳证据。

## 二、已接入能力

网页顶部“工作台”打开独立弹窗，不与设置面板混在一起。

工作台支持选择当前对话或历史运行记录，并按树状结构展示：

```text
创作流程
├─ 输入与理解
│  ├─ 对话记录
│  ├─ 输入材料
│  └─ 素材理解 v1 / v2 ...
├─ 剧本生成与检查
│  ├─ 剧本 v1 / v2 ...
│  └─ 剧本检查 v1 / v2 ...
├─ 分镜生成与检查
│  ├─ 分镜 v1 / v2 ...
│  └─ 分镜检查 v1 / v2 ...
└─ 归档学习
   └─ 手动归档记录
```

点击节点后，在工作台内查看对应 Markdown 内容。

## 三、手动归档学习

自动无感学习仍然保留。网页对话中出现偏好、规则、质量反馈、流程要求等信号时，仍会自动写入学习记录。

手动归档学习是新增的高置信度人工证据，不替代自动学习。

手动归档流程：

1. 在思维导图中点击一个产物节点。
2. 点击“加入归档”。
3. 在采纳清单里填写采纳范围和采纳原因。
4. 点击“手动归档学习”。

手动归档不是简单选择某一个版本，而是整理“采纳清单”。例如：

- 第 1-2 集采纳剧本 v1。
- 第 3-4 集采纳剧本 v2。
- 某条镜头节奏规则采纳分镜检查 v2。
- 某段对白采纳人工修改后的节点。

归档结果会写入：

- `learning/conversation-records/`
- 对应 `runs/运行目录/`

这样后续学习闭环可以区分来源：自动无感学习是 `conversation` 证据，手动归档是 `manual-archive` 证据。两者可能重复，但人工归档的置信度更高。

## 四、聊天气泡复制

每条聊天气泡下方新增“复制”按钮。

用途：

- 快速复制用户输入的提示词。
- 快速复制助手生成的剧本、分镜或评审结果。
- 避免为了重复生成而在工作台里再做一个输入区。

## 五、后端接口

新增或复用的网页接口：

- `GET /api/runs`：读取历史运行记录。
- `GET /api/workbench?run=运行目录`：返回流程树、阶段状态和可归档节点。
- `GET /api/artifact?run=运行目录&file=文件名`：读取节点正文。
- `POST /api/archive-learning`：把采纳清单写成手动归档学习记录。

流程树和归档 Markdown 由 `app/lib/workbench.js` 统一维护，避免前端和后端各写一套节点规则。
