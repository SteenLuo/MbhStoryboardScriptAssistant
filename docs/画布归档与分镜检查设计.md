# 画布归档与分镜检查设计

日期：2026-07-01

本文定义画布归档并学习、分镜生成后检查、红标草稿、轻量 Markdown 编辑和画布入口补齐。它是自主学习机制的配套设计。

## 一、画布归档并学习

归档不是删除。归档后的画布进入只读存档区，仍可查看、复制、删除到回收站。删除不是物理删除，而是移入回收站。

归档前必须郑重提示：

1. 归档后不可编辑。
2. 只能查看和复制。
3. 如需继续修改，只能复制为新画布。
4. 同一事物必须保持唯一版本，方便后台学习。

归档后启动慢速学习任务。学习任务不能阻塞用户继续使用软件。

## 二、唯一版本核对

归档前系统识别学习槽位：

- 小说
- 剧本
- 分镜第 1 集
- 分镜第 2 集
- 其他分镜集数

如果每个槽位只有一个当前可用版本，直接允许归档，并显示“已满足唯一版本要求”。

只有同一槽位存在多个当前可用版本时，才要求用户强制选择唯一版本。

判定规则：

1. 合并节点只统计主版本，历史版本不参与重复判定。
2. 修改节点或重生成节点如果不是当前采用版本，不应误判为重复。
3. 系统无法准确判断时进入核对界面，让用户确认，不能直接报错拦死。
4. 归档判定错误导致唯一版本被误拦，是严重 bug。

## 三、归档入口

左下角入口：

- 回收站图标
- 已归档项目图标
- 学习资料库图标
- 设置图标

画布列表右键菜单：

- 重命名
- 删除
- 归档

画布顶部标题附近：

- 删除
- 归档

已归档项目入口展示只读归档画布。可执行：

- 查看
- 复制为新画布
- 删除到回收站

不允许解冻原归档。

## 四、分镜生成后检查

第一版先覆盖分镜，重点检查：

1. 台词每句 20 字以内，超出应拆新分镜。
2. 字段完整。
3. 镜号连续，不重复、不跳乱。
4. 禁止制作备注、导演备注等不应交付内容。

生成流程：

1. 后台生成分镜。
2. 运行生成后检查。
3. 有问题时调用模型二次修正。
4. 修正通过则正常生成节点。
5. 修正后仍有问题，则生成红标草稿。

多集批量生成时，不要求整批全过才显示。自动修正后仍有问题的集数以红标草稿出现，避免用户没有编辑入口。

## 五、红标草稿

节点级提示：

- 发现 N 处提示
- 已知晓
- 仍然采用
- 已修正

用户动作：

1. 我已知晓：用户已看到系统提示。若后续无修改，可以继续作为当前可用版本；若有修改或重生成，以最终版本为准。
2. 仍然采用：用户明确决定当前这一版继续使用。它不是系统检查通过，也不作为正向学习证据。
3. 手动编辑保存：视为接受当前结果。保存后可重新检查，检查通过则转正常，仍有问题则保留提示记录。
4. 修改功能重新生成：新版本重新走检查，旧版本保留历史。

## 六、问题详情定位

点击问题详情后，全屏展示文字节点。

主区域显示完整文本，侧边栏列出具体问题：

- 镜号
- 字段
- 问题原因
- 原文片段
- 修改建议
- 行号或字符范围

点击问题项，主区域滚动到对应行或片段并高亮。

检查结果保存到节点元数据，不写进正文：

- `issueId`
- `severity`
- `shotNo`
- `field`
- `message`
- `excerpt`
- `suggestion`
- `lineStart`
- `lineEnd`
- `charStart`
- `charEnd`

## 七、轻量 Markdown 编辑

画布节点内容支持轻量 Markdown。它是展示和编辑增强，不负责问题状态。

支持：

- H1/H2/H3
- 正文
- 加粗
- 斜体
- 有序列表
- 无序列表
- 分割线
- 复制文本
- 展开编辑或全屏编辑
- 节点背景色

不支持：

- 表格
- 图片嵌入
- 任意 HTML
- 复杂字体颜色
- 所见即所得富文本拖拽编辑

工具条行为：

1. 平时画布节点不显示工具条。
2. 单击节点只负责选中节点，不进入文本编辑态。
3. 双击节点正文时，必须在原节点框内编辑，并在节点上方浮出工具条。
4. 全屏编辑不是双击默认行为，只能通过节点上方工具条的“全屏”入口进入。
5. 展开或全屏编辑时，工具条固定在编辑区顶部。
6. 节点框内编辑区和全屏编辑区都必须使用项目配色的细滚动条，不允许回退为浏览器原生滚动条。
7. 轻量 Markdown 编辑不显示“保存”按钮；标题或正文发生编辑后自动保存。
8. 关闭按钮只负责关闭窗口，不承担保存语义；关闭前仍需强制落盘当前未写入的编辑内容。
9. 一次只显示当前正在编辑的工具条。

节点背景色属于用户内容组织。系统红标使用边框、问题条和问题清单，不占用背景色语义。
