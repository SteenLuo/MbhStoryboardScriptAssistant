# 技能系统设计

## 一、目标

建立一套本地技能系统，用来把不同类型、不同题材的小说改编成 **AI漫剧剧本** 和 **AI漫剧分镜**。

第一层功能始终只有两个：

1. 生成 AI 漫剧剧本。
2. 生成 AI 漫剧分镜。

小说类型、题材、受众和平台只是路由条件，不是额外产品功能。

默认平台语境是 AI 漫剧。除非用户明确指定其他用途，否则所有格式、评审和学习都按 AI 漫剧生产来判断。

## 二、借鉴规则

本系统选择性借鉴：

- 从 `gstack` 借鉴专业角色、评审关卡和产物交接。
- 从 `gbrain` 借鉴记忆、评测样例和技能优化。
- 从 `0WcAssistant` 借鉴本地技能蓝图，以及“稳定技能从真实重复动作里长出来”的原则。

本系统不借鉴：

- 广义个人助理驾驶舱。
- 今日摘要。
- 无关模块接入系统。
- 大型个人知识图谱。
- 不经审查的自动改写技能。

## 五、核心技能

三种应用场景对应不同技能链路：

| 使用场景 | 起点 | 必经技能 | 可跳过技能 |
| --- | --- | --- | --- |
| 小说生成剧本，再生成分镜 | 小说原文 | 输入整理、类型判断、剧本生成、剧本评审、分镜生成、分镜评审、学习记录 | 无 |
| 提供剧本，评审改写后生成分镜 | 剧本 | 剧本分析、剧本评审、剧本改写、分镜生成、分镜评审、学习记录 | 小说输入整理 |
| 直接提供剧本生成分镜 | 已认可剧本 | 剧本节拍识别、分镜生成、分镜评审、学习记录 | 小说输入整理、剧本改写 |

### 1. 小说输入整理

作用：把原始小说输入整理成稳定的基础信息和来源简报。

输入：

- 小说正文或章节片段。
- 目标平台。
- 目标时长或字数。
- 用户补充要求。

输出：

- 基础信息文件。
- 梗概文件。
- 小说总纲。
- 每集大纲。
- 每集 2 到 4 个爽点或剧情高潮点。
- 单集和整体剧情负面逻辑检查。
- 风险提示。
- 待确认问题。

### 2. 类型和题材判断

作用：选择合适的改编规则。

初始类型桶：

- 都市情感
- 古风情感
- 悬疑惊悚
- 推理
- 玄幻
- 仙侠 / 修仙
- 科幻
- 家庭伦理
- 职场故事
- 复仇 / 逆袭
- 喜剧
- 青春 / 校园

判断结果应包括：

- 主类型
- 副类型
- 题材标签
- 语气标签
- 目标受众
- 节奏模型
- 可能适用的剧本公式
- 分镜视觉风格提示

### 3. 剧本生成

作用：生成适合 AI 漫剧生产的场景化剧本。

当前落地技能：`skills/02-script/script-generate`。

必须包含：

- 标题
- 原文范围
- 改编前提
- 场景列表
- 分场剧本
- 旁白
- 对白
- 动作
- 改编说明
- 未解决风险

类型规则可以不同，但最终格式要稳定。

核心要求：

- 开场即冲突。
- 每场结尾有钩子。
- 爽点和情绪持续推进。
- 动作描写适合 AI 视频生成。
- 保留原著核心剧情、人物关系、关键伏笔和爽点逻辑。

### 4. 剧本评审

作用：在生成分镜前拦住弱剧本。

当前落地技能：`skills/02-script/script-review-rewrite`。

评审维度：

- 是否忠于原文核心
- 冲突是否清楚
- 开头是否有钩子
- 场景节奏是否合理
- 角色动机是否成立
- 对白是否有用
- 是否适配目标平台
- 连贯性是否稳定
- 是否缺上下文
- 是否过度压缩
- 每集是否有 2 到 4 个爽点或剧情高潮点
- 单集剧情是否存在逻辑不通
- 整体剧情是否存在跨集矛盾

输出：

- 通过 / 需要修改
- 按优先级排列的问题
- 对应场景
- 推荐修改方式

### 5. 分镜生成

作用：把通过评审的剧本转成适合 AI 画面生成和 AI 视频生成的视觉节拍。

每条分镜建议包含：

- 镜头编号
- 场景编号
- 这一镜头的叙事目的
- 景别
- 镜头运动
- 画面构图
- 角色动作
- 对白或旁白
- 时长估计
- 连贯性备注
- 制作备注

AI 漫剧分镜还要特别注意：

- 主体清楚，避免一个镜头里塞入过多人物和动作。
- 画面描述具体，便于后续转成 AI 画面提示词或视频提示词。
- 动作连续但不要过度复杂，降低 AI 生成失败率。
- 情绪、字幕、音效和镜头目的要能一一对应。

### 6. 分镜评审

作用：发现视觉表达和连续性问题。

评审维度：

- 是否覆盖剧本
- 画面是否清楚
- 镜头是否有变化
- 连续性是否成立
- 镜头是否可拍
- 角色位置是否清楚
- 是否缺少过渡
- 是否重复镜头过多
- 制作说明是否含糊

分镜评审必须给出去向判断：

| 评审结论 | 问题来源 | 后续流程 |
| --- | --- | --- |
| 通过 | 分镜可用 | 进入分镜定稿 |
| 需要小修 | 分镜表达层问题 | 分镜局部修订 |
| 需要重生成 | 拆镜策略问题 | 重新生成对应场次或整段分镜 |
| 返回剧本 | 剧本源头问题 | 回到剧本评审 / 改写 |

判断原则：

- 如果只是字段漏填、镜头描述空泛、景别运镜不准，修分镜。
- 如果整场镜头节奏、顺序、覆盖关系都有问题，重生成分镜。
- 如果剧本本身缺动作、缺因果、冲突抽象、无法画面化，返回剧本评审。

### 7. 学习记录

作用：把真实使用中的问题转成未来改进候选。

示例记录：

```json
{"type":"问题","genre":"悬疑惊悚","stage":"剧本","problem":"开头钩子太慢","evidence":"用户重写了前三场","candidate_rule":"悬疑开头应在前两场出现威胁、异常事件或不可逆选择。"}
```

## 六、自我完善模型

自我完善不等于每次运行后立刻改正式技能。

采用带门槛的流程：

1. 捕获反馈和失败。
2. 聚合同类问题。
3. 提出技能修改建议。
4. 增加或更新评测样例。
5. 对比修改前后的效果。
6. 只有确认改善后，才接受技能修改。

学习分三层：

1. **运行学习**：把反馈保存到本次运行目录。
2. **候选规则**：重复问题进入候选规则文件。
3. **正式规则**：候选规则通过审查和评测后，再进入正式技能或正式参考文件。

这样可以让系统不断进化，又避免一次坏样例污染技能。

## 七、已有样例学习

已有剧本和分镜是本项目最重要的训练材料之一。它们不直接改写正式技能，而是先用来提炼：

- 剧本格式习惯
- 分镜表字段习惯
- 镜头密度
- 场景拆分方式
- 常见旁白和对白写法
- 不同题材的视觉风格
- 用户修改偏好
- 可沉淀的候选规则

样例学习流程：

1. 把已有剧本和分镜放入 `samples/`。
2. 为每组样例补充一个说明文件，说明来源、题材、是否满意、哪里值得学习。
3. 系统读取样例，提炼格式和风格。
4. 提炼结果先进入候选规则。
5. 经过人工确认后，再进入正式技能或参考文件。

## 八、评测样例类型

### 1. 类型判断评测

检查系统是否能选对小说类型和题材。

示例：

```json
{"task_id":"route-urban-revenge-001","input":"女主被家族陷害，五年后隐姓埋名归来，开始反击当年害她的人。","expected_genre":"复仇 / 逆袭","expected_tags":["隐藏身份","家族冲突","归来"]}
```

### 2. 剧本生成评测

检查剧本是否包含必要结构，并符合类型期待。

可检查：

- 是否有场景列表
- 是否有对白
- 是否有动作
- 是否有改编说明
- 开头是否有钩子
- 冲突是否升级
- 是否编造了原文不支持的结局

### 3. 分镜生成评测

检查分镜是否具有视觉可用性。

可检查：

- 每条分镜是否有景别
- 每条分镜是否对应场景
- 是否有镜头说明
- 是否对应对白或旁白
- 是否存在空泛画面描述
- 转场是否明确

## 九、第一步要建的技能

先建一个总控技能：

```text
novel-to-script-storyboard
```

它先覆盖少量类型，跑通完整流程。等产物协议稳定后，再拆成专业技能。

推荐第一批类型：

1. 都市情感
2. 悬疑惊悚
3. 玄幻 / 仙侠
4. 家庭伦理

这几类的节奏、视觉和对白差异较大，适合作为第一批验证对象。
