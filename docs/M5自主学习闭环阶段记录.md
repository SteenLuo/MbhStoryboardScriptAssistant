# M5 自主学习闭环阶段记录

日期：2026-06-10

## 一、当前结论

M5 已完成可验收版本。

当前项目已经具备“越用越聪明”的基础闭环，但不是完全自动改技能。M5 的完成标准是可追溯、可审查、可回退，而不是让系统不经证据自动修改正式技能。

补充原则：越用越聪明不等于越学越多。发现学习后质量下降时，系统必须能及时止损、记录降质、冻结可疑规则或提出回退建议。

正确理解是：

```text
投放样例
-> 自动扫描
-> 自动生成分拣报告
-> 自动生成 M4 任务草案
-> 未知材料先读正文二次判断
-> 进入 M4 评测或对齐复核
-> 对话结束前判断是否有可学习信息
-> 如有新偏好、流程要求或质量反馈，写入对话学习记录
-> 生成学习快照
-> 如发现质量下降，记录降质和回退风险
-> skill-evolution 基于证据提出技能改进建议
-> 满足条件后再修改正式技能
```

## 三、仍不能自动做的事

当前系统还不能完全自动：

- 判断所有 `.doc`、`.docx` 历史文件正文中的隐含版本关系。
- 自动确认某份历史分镜是否是最终认可版本。
- 自动把题材专属规则升级为正式规则。
- 自动替用户决定“某个偏好是否长期有效”。
- 自动判断所有质量下降的根因；复杂情况仍需要通过 M5 回归评测确认。

这些仍需要证据积累，必要时向使用人员提出简短问题。

## 四、当前规则判断

### 可以较稳定使用的基础规则

- 分镜字段必须完整。
- 历史分镜有缺字段、乱码、镜号跳号时，新输出必须修正。
- 部分对应样例只能学习可靠片段。
- 一部剧可以是一对多样例关系。
- 分镜生成后必须做目标时长校准。
- 对话场要有反应、物品和信息层切镜。

### 继续候选的题材规则

- 经营逆袭镜头链。
- 身份反转镜头链。
- 修仙开篇世界观混剪。
- 修仙打脸爽点链。
- 修仙资源反转链。
- 豪门替嫁开篇冲突链。
- 强女主反击画面化。

原因：

- 它们目前多来自同一部剧或同一题材组。
- 可以作为生成和评测参考。
- 暂不应升级为所有题材默认规则。

## 五、下一步建议

1. 后续重点不是文件夹监听，而是把“对话结束学习判断”固化为默认工作习惯。
2. 继续收集跨剧目的经营、身份反转、修仙、豪门强女主样例。
3. 每次新样例投放后固定执行：

```text
New-InboxScanReport
-> New-M4TaskDraft
-> unknown-material-review
-> M4 对齐 / 覆盖复核
-> New-LearningSnapshot
-> 如有降质，New-QualityRegressionRecord
-> skill-evolution 建议
```

4. 每次对话结束前固定判断：

```text
普通问答 -> 静默跳过
新偏好 / 流程要求 / 样例解释 / 质量反馈 -> New-ConversationLearningRecord
质量下降 -> New-ConversationLearningRecord + New-QualityRegressionRecord
```

5. 暂不自动升级题材专属正式规则，除非出现跨剧目证据或用户明确确认。
