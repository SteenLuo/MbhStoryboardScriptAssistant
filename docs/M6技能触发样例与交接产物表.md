# M6 技能触发样例与交接产物表

日期：2026-06-11

## 一、总原则

M6 的目标不是增加新功能，而是把已有能力拆成可维护的专业技能，并让总控入口能正确路由。

每个技能只负责一类事情：

- 输入整理只做信息结构化和正负分析。
- 剧本生成只负责从材料到剧本。
- 剧本评审只负责判断剧本是否能进入分镜，并给出改写方向。
- 分镜生成只负责从剧本到分镜，并做分镜评审去向。
- 样例学习只负责从历史样例提炼候选规则。
- 技能进化只负责把证据转成可审查、可回退的技能改进建议。

## 二、技能触发表

| 技能 | 触发样例 | 输入 | 输出 | 下游 |
| --- | --- | --- | --- | --- |
| `mbh-workflow` | “这是小说，帮我做成剧本和分镜” | 用户自然语言、小说、剧本、分镜、反馈 | 路由判断、运行目录建议、是否学习判断 | 所有专业技能 |
| `novel-intake` | “先整理这段小说的大纲和爽点” | 小说原文、章节、梗概、人设 | 小说大纲、每集大纲、爽点、负面逻辑检查 | `script-generate` |
| `script-generate` | “根据小说生成 AI 漫剧剧本” | 输入分析、小说大纲、爽点、用户要求 | AI 漫剧剧本草稿 | `script-review-rewrite` |
| `script-review-rewrite` | “评审这个剧本能不能进入分镜” | 剧本草稿或已有剧本 | 通过 / 轻改 / 重写、问题清单、修订剧本 | `storyboard-generate` |
| `storyboard-generate` | “按这个剧本生成分镜” | 已通过剧本、分镜要求、标准格式 | 标准分镜、分镜评审、小修建议 | `skill-evolution` |
| `sample-ingest` | “这些剧本和分镜样例学习一下” | `samples/` 文件、样例说明、对话中的样例关系解释 | 分拣报告、关联关系、候选规则、待确认问题 | `skill-evolution` |
| `skill-evolution` | “这个规则补进技能里” / “学习后质量下降” | 运行记录、评测、候选规则、对话学习记录、降质记录 | 技能进化草案、回退建议、正式规则建议 | 对应技能文件 |

## 三、三条主链路交接

### 小说到剧本再到分镜

```text
mbh-workflow
-> novel-intake
-> script-generate
-> script-review-rewrite
-> storyboard-generate
-> skill-evolution
```

交接产物：

| 上游 | 下游 | 文件 / 内容 |
| --- | --- | --- |
| `novel-intake` | `script-generate` | 输入分析、小说大纲、每集大纲、爽点、负面分析 |
| `script-generate` | `script-review-rewrite` | 生成剧本 |
| `script-review-rewrite` | `storyboard-generate` | 通过评审或修订后的剧本 |
| `storyboard-generate` | `skill-evolution` | 分镜、分镜评审、覆盖核对、用户反馈 |

### 剧本评审改写再到分镜

```text
mbh-workflow
-> script-review-rewrite
-> storyboard-generate
-> skill-evolution
```

交接产物：

| 上游 | 下游 | 文件 / 内容 |
| --- | --- | --- |
| `script-review-rewrite` | `storyboard-generate` | 轻改剧本或通过评审剧本 |
| `storyboard-generate` | `skill-evolution` | 分镜产物和评审结果 |

### 认可剧本直接到分镜

```text
mbh-workflow
-> storyboard-generate
-> skill-evolution
```

交接产物：

| 上游 | 下游 | 文件 / 内容 |
| --- | --- | --- |
| `mbh-workflow` | `storyboard-generate` | 已认可剧本和分镜要求 |
| `storyboard-generate` | `skill-evolution` | 分镜、评审、反馈 |

## 四、学习和进化交接

| 来源 | 进入位置 | 产物 | 后续 |
| --- | --- | --- | --- |
| 新样例文件 | `sample-ingest` | 分拣报告、M4 任务草案 | 学习快照、技能进化草案 |
| 对话偏好 | `New-ConversationLearningRecord.ps1` | 对话学习记录 | 对话候选规则草案 |
| 学习后降质 | `New-QualityRegressionRecord.ps1` | 降质记录 | M5 回归评测任务 |
| 评测通过的重复规则 | `skill-evolution` | 技能进化草案 | 修改正式技能或继续候选 |

## 五、验收标准

- 每个技能都有独立触发条件。
- 每个技能都有明确输入和输出。
- 技能之间通过文件或结构化内容交接。
- 总控技能只负责路由，不替代专业技能。
- M7 网页可以只呈现聊天入口，但后端仍按本表进行技能路由。
