# 里程碑和节点

说明：本文是早期总路线，保留完整阶段设想。当前执行进度和最新阶段命名以 [当前里程碑节点](当前里程碑节点.md) 为准。当前执行版中，“样例学习”已作为 M2 推进，三条业务链路已作为 M3 完成试跑。

## M0：项目基础

目标：明确项目边界、产物协议和参考架构。

节点：

- M0.1 创建项目说明文档。
- M0.2 明确两个核心功能：生成 AI 漫剧剧本、生成 AI 漫剧分镜。
- M0.3 定义每次运行的产物目录。
- M0.4 结合 `gstack`、`gbrain` 和 `0WcAssistant` 讨论，形成选择性借鉴原则。
- M0.5 明确本地运行产物和技能文件放在哪里。
- M0.6 明确本项目不借鉴 `0WcAssistant` 的哪些方向。
- M0.7 写出综合参考判断。

验收标准：

- 文档已经存在。
- 一分钟内能讲清楚项目做什么。
- 后续功能都能用“是否服务 AI 漫剧剧本和分镜”来判断。
- 项目不会被误解成通用个人助理或驾驶舱。
- 有一个主文档统一承载参考判断，避免决策分散。

## M1：本地分层技能原型

目标：创建第一批可用的本地分层技能，完成小说到 AI 漫剧剧本再到分镜的完整流程。

节点：

- M1.1 创建 `skills/` 分层目录。
- M1.2 创建输入整理技能：`novel-intake`。
- M1.3 创建 AI 漫剧剧本生成技能：`script-generate`。
- M1.4 创建剧本评审改写技能：`script-review-rewrite`。
- M1.5 创建 AI 漫剧分镜生成技能：`storyboard-generate`。
- M1.6 创建样例学习技能：`sample-ingest`。
- M1.7 创建技能进化技能：`skill-evolution`。
- M1.8 每个技能补充中文说明、触发场景、输入、输出和质量门槛。
- M1.9 每个技能至少有一个参考规则文件。
- M1.10 支持三种入口：小说入口、剧本评审改写入口、剧本直接分镜入口。

验收标准：

- 技能按职责分层，不平铺。
- AI 漫剧剧本生成有独立技能承接。
- 输出结构稳定。
- 能记录人工反馈。
- 三种应用场景都有明确入口。

## M1.5：已有样例学习

目标：把已有剧本和分镜积累整理成可学习材料。

节点：

- M1.5.1 创建 `samples/` 样例目录。
- M1.5.2 定义剧本样例和分镜样例的放置格式。
- M1.5.3 为每个样例补充说明文件。
- M1.5.4 从样例中提炼剧本格式、分镜字段、镜头密度和风格偏好。
- M1.5.5 把提炼结果写入候选规则。

验收标准：

- 至少 3 组已有剧本 / 分镜样例被整理。
- 能说明每组样例值得学习什么。
- 样例不会直接污染正式技能，只进入候选规则。

## M2：评审关卡

目标：避免弱剧本直接进入分镜生成。

节点：

- M2.1 创建剧本评审规则。
- M2.2 创建分镜评审规则。
- M2.3 增加“通过 / 需要修改”的判断。
- M2.4 评审问题必须指向具体场景或镜头。
- M2.5 分镜生成必须使用通过评审或已修订的剧本。
- M2.6 增加分镜评审后的去向判断：分镜小修、重新生成分镜、返回剧本评审。

验收标准：

- 剧本评审能发现节奏、动机、忠实度问题。
- 分镜评审能发现画面缺失、镜头重复、连续性问题。
- 评审结果能直接指导修改，而不是泛泛而谈。
- 分镜不通过时，不会停在死路，而是进入明确后续流程。

## M3：类型和题材判断

目标：让系统对不同小说类型采取不同生成策略。

节点：

- M3.1 定义小说类型分类。
- M3.2 定义题材标签和语气标签。
- M3.3 定义类型判断输出格式。
- M3.4 增加类型专属剧本规则。
- M3.5 增加类型专属分镜规则。
- M3.6 每个初始类型至少创建 5 条判断样例。

验收标准：

- 系统能判断主类型和副类型。
- 不同类型的剧本输出会有差异。
- 不同类型的分镜输出会有差异。

## M4：学习记录

目标：保存每次运行发生了什么，让反复使用能改善未来表现。

节点：

- M4.1 定义学习记录格式。
- M4.2 记录用户修改和评审发现。
- M4.3 在运行清单中记录技能版本和提示词版本。
- M4.4 增加定期学习复盘文档或命令。
- M4.5 把重复问题转成候选规则。
- M4.6 候选规则和正式规则分开保存。

验收标准：

- 每次运行都能说明由哪个技能版本生成。
- 反馈能变成结构化记录。
- 重复失败能被看见，而不是埋在聊天记录里。
- 单次运行不能静默改写正式技能。

## M5：评测样例驱动的技能改进

目标：让技能改进可衡量。

节点：

- M5.1 创建类型判断评测样例。
- M5.2 创建剧本生成评测样例。
- M5.3 创建分镜生成评测样例。
- M5.4 增加结构完整性的确定性检查。
- M5.5 增加类型质量检查。
- M5.6 接受技能修改前，对比新旧版本效果。
- M5.7 记录已接受修改的前后证据。

验收标准：

- 技能修改会经过样例测试。
- 项目能拒绝“听起来更好但实际更差”的修改。
- 改进有证据。
- 自我完善可审查、可回退。

## M6：拆分专业技能

目标：从一个总控技能演进成可维护的技能包。

节点：

- M6.1 拆分输入整理、类型判断、剧本生成、剧本评审、分镜生成、分镜评审、学习记录。
- M6.2 为每个技能增加触发样例。
- M6.3 定义技能之间的交接产物。
- M6.4 增加一个技能路由说明文档。
- M6.5 保留总控技能作为简单入口。

验收标准：

- 每个技能只做一件事。
- 技能可以独立改进。
- 完整流程仍然能跑通。

## M7：本地工具或界面

目标：把工作流包成可重复使用的本地入口。

节点：

- M7.1 决定先做命令行还是本地网页。
- M7.2 实现输入收集。
- M7.3 实现运行目录创建。
- M7.4 实现产物查看。
- M7.5 实现反馈记录。
- M7.6 支持基于已有产物重新运行。

验收标准：

- 用户不需要手动在多个文件之间复制。
- 输出能稳定保存。
- 反馈能在评审时记录下来。

## M8：稳定使用

目标：让系统能支撑长期真实改编工作。

节点：

- M8.1 增加技能和运行产物的备份 / 导出。
- M8.2 处理长篇小说输入。
- M8.3 增加章节切分和连续性记忆。
- M8.4 增加失败恢复。
- M8.5 增加近期运行质量看板。
- M8.6 增加“如何新增小说类型”的文档。
- M8.7 增加技能草案收件箱：让使用人员在网页内接收技能进化草案、候选规则草案和评测任务草案，并完成采纳、驳回、继续观察和二次确认应用到 `SKILL.md`。

验收标准：

- 长文本能安全处理。
- 类型技能能扩展，不需要重写系统。
- 用户能查看近期质量变化。
- 技能修改草案不只停留在本地文件里，必须有网页端确认入口，且正式改写 `SKILL.md` 前要经过二次确认。
