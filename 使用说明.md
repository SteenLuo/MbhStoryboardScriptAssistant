# 客户试用使用说明

## 一、第一次使用

1. 解压客户试用包。
2. 双击根目录的 `启动助手.bat`。
3. 如果电脑没有 Node.js，脚本会下载并打开 Node.js 安装包。请完成安装后，再次双击 `启动助手.bat`。
4. 浏览器打开后，进入右下角或左侧的“设置”。
5. 选择模型供应商。默认是 DeepSeek，也可以选择 OpenAI（GPT 5.5）。
6. 填写对应供应商的 API Key，并点击“保存配置”。
7. 点击“测试连接”，确认能正常连接。

## 二、日常启动

以后使用时，双击：

```text
启动助手.bat
```

如果服务已经在运行，脚本只会打开浏览器页面。

如果服务没有运行，脚本会先启动服务，再打开浏览器页面。

默认地址：

```text
http://127.0.0.1:17877
```

## 三、关闭服务

使用结束后，双击：

```text
关闭服务.bat
```

它会关闭本项目启动的本地服务。

## 四、开始使用：对话模式

你可以直接在对话框里输入：

- 小说原文
- 已有剧本
- 分镜需求
- 修改意见
- 样例学习要求
- 对输出结果的反馈

也可以点击输入框旁边的上传按钮，上传文本文件或图片。

## 五、项目分组

左侧对话列表支持按项目分组管理。

- 新建对话时可以选择已有项目，也可以创建新项目。
- 项目可以折叠和展开，便于把同一个剧目的对话放在一起。
- 旧对话如果没有项目，会归到默认的“无项目”分组。
- 项目分组只影响本机管理和查找，不会改变每个对话自己的上下文。

## 六、画布模式

页面支持切换到画布工作区。画布适合把一个剧目的小说、剧本、分镜和备注材料放在同一张图里管理。

画布里常用节点包括：

- 小说 / 原始材料节点
- 剧本节点
- 分镜脚本节点
- 备注节点

常用操作：

- 双击或打开节点编辑内容。
- 拖动画布节点可以调整位置。
- 节点两侧加号可以创建关联节点或拖拽连线。
- 小说节点点击加号后，可以生成剧本节点。
- 剧本节点点击加号后，可以自动识别 `第1集`、`第2集` 等分集，并批量生成对应集数的分镜脚本节点。
- 画布搜索会在当前画布内查找节点标题和内容。

画布保存到本机 `app/data/canvases/`。这是运行数据，不会包含在维护人员发出的干净客户包里。

## 七、流程思维导图

点击顶部“工作台”，会打开独立弹窗。

工作台可以：

- 选择当前对话或历史运行记录。
- 用树状图查看输入、理解、剧本版本、剧本检查、分镜版本、分镜检查和归档学习。
- 点击节点查看对应内容。
- 把被采纳的节点加入采纳清单。
- 点击“手动归档学习”，把本次人工确认过的采纳证据写入学习记录。

自动无感学习仍然会保留。手动归档学习只是更精准的人工精选证据，不会直接覆盖正式技能规则。

聊天气泡下方有“复制”按钮，可以复制你的提示词，也可以复制助手生成的剧本、分镜或评审结果。

## 八、发送验证

- 输入框没有文字、也没有附件时，发送按钮不可用。
- 输入文字后，发送按钮会自动变为可用。
- 添加附件后，即使没有文字，也可以发送。
- 按 Enter 发送时也会做同样验证，不会发送空消息。
- Shift+Enter 可以换行。

## 九、对话窗口和回收站

- 左侧是多个对话窗口。
- 每个窗口互不串上下文。
- 右键对话窗口可以重命名或删除。
- 删除只是从网页列表隐藏，不会删除本地文件。
- 左下角“回收站”可以恢复被隐藏的窗口。

## 十、关于 skills

如果你收到的是 `Full` 包：

- 包内包含初始 `skills/`。
- 适合第一次使用。

如果你收到的是 `NoSkillOverwrite` 包：

- 包内不包含 `skills/`。
- 适合已经使用过一段时间、有自己技能沉淀的版本更新。
- 这样不会覆盖你本地已经进化过的 `skills/`。

## 十一、不要手动删除的目录

正常使用时，不建议手动删除：

- `app/data/`
- `runs/`
- `learning/`
- `skills/`

这些目录可能保存你的对话、产物、学习记录和技能沉淀。

## 十二、隐私说明

客户试用包不会包含维护人员本机的：

- DeepSeek / OpenAI 等模型供应商 API Key
- 历史对话窗口
- 测试运行记录
- 本机日志

你在本机填写的 API Key 会保存在你自己的本地目录中。
