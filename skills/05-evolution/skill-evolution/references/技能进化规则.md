# 技能进化规则

## 候选规则进入正式规则的条件

至少满足以下条件之一：

- 三次以上真实运行出现同类问题。
- 两组以上高质量样例支持同一规则。
- 用户明确确认这是长期偏好。
- 评测样例显示修改后更好。

## 升级优先级

优先升级跨题材基础规则：

- 输出格式和字段完整性。
- 单镜时长和总时长校准。
- 运镜和构图的通用限制。
- 样例关系处理，例如一对多、部分对应、历史噪声修正。

谨慎升级题材专属规则：

- 经营逆袭。
- 身份反转。
- 修仙 / 玄幻。
- 豪门替嫁 / 强女主。

题材专属规则如果只来自同一部剧，应继续候选，不要直接写入正式技能默认规则。

## 必须继续观察的情况

- 只有一次反馈。
- 规则只适用于一个特殊作品。
- 与已有规则冲突。
- 会降低其他类型小说效果。

## 降质和回退规则

如果用户反馈学习后输出质量下降，或系统评审发现学习导致明显退步，必须先止损。

常见降质信号：

- 剧本更不连贯，人物动机更弱。
- 分镜字段缺失、镜号混乱、时长明显偏离。
- 运镜变多但不服务剧情。
- 对话切镜过碎，画面跳跃。
- 题材规则被过度套用，导致不同题材变成同一种味道。
- 为了学习历史样例，反而继承了历史样例的缺字段、乱码、制作备注。
- 用户明确说“不如之前”“学偏了”“质量下降”。

处理顺序：

1. 先写入 `learning/regression-reports/`。
2. 暂停把本次输出作为正向样例。
3. 找出可能关联的候选规则、已确认规则或技能修改。
4. 如果关联候选规则，将其状态改为暂停观察或继续观察。
5. 如果关联已确认规则，不直接删除，先生成技能进化草案，建议收窄适用边界或增加反例。
6. 如果连续两次以上出现同类降质，建立 M4 回归评测任务。
7. 只有回归评测通过后，才允许继续强化该规则。

## 对话学习记录规则

对话学习记录用于保存用户在使用过程中的自然反馈，不要求用户主动说“学习”。

需要记录的情况：

- 用户补充了新的长期偏好或流程要求。
- 用户解释了某个样例之间的真实关系。
- 用户指出输出哪里好、哪里不好。
- 用户给出新的题材规律、分镜要求、剧本要求或反例。
- 用户反馈学习后质量下降。

不需要记录的情况：

- 普通状态询问。
- 临时确认、寒暄或没有项目增量的问答。
- 已经被现有文档明确覆盖，且没有新增边界的信息。

处理顺序：

1. 先判断是否值得学习；不值得学习时静默跳过。
2. 值得学习但证据不足时，写入 `learning/conversation-records/`，标为候选或待评测。
3. 如果对话中出现质量下降信号，同时写入 `learning/regression-reports/`。
4. 对话记录只作为证据入口，不直接改正式 skill。
5. 如果同类对话反馈反复出现，再由技能进化草案建议进入候选规则或正式规则。

## 版本记录内容

每次正式修改技能后，记录：

- 修改日期。
- 修改技能。
- 修改原因。
- 证据来源。
- 修改前问题。
- 修改后预期。
- 回退方式。

## 证据引用要求

技能进化报告应尽量引用：

- `learning/snapshots/` 中的最新学习闭环快照。
- `learning/conversation-records/` 中的对话学习记录。
- `learning/regression-reports/` 中的降质记录。
- `runs/` 中的运行目录。
- `learning/evals/` 中的评测结果。
- `learning/candidate-rules/` 中的候选规则编号。
- `learning/accepted-rules/` 中的已确认规则编号。

## 学习快照使用规则

提出技能改进建议前，先读取最新学习快照。

学习快照用于判断：

- 当前已有多少运行证据。
- 是否已有对齐复核、覆盖核对和运行总结。
- 候选规则和已确认规则入口是否齐全。
- 最新一轮 M4 任务是否已经完成。

快照只提供索引，不代替正文证据。需要修改正式技能时，仍要回到对应 `runs/`、`learning/candidate-rules/` 和 `learning/accepted-rules/` 读取原始证据。
