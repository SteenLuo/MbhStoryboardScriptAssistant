# 样例学习规则

## 可以提炼的内容

- 剧本结构。
- 分镜字段。
- 镜头密度。
- 景别偏好。
- 旁白和对白比例。
- AI 漫剧画面生成友好度。
- AI 漫剧字幕、音效和镜头节奏习惯。
- 用户修改前后差异。
- 不同题材的特殊表达。

## 原始投放区处理规则

`samples/_inbox/` 是混乱材料投放区。`samples/` 根目录下的散落文件也视为混乱材料。处理时先分拣，再学习。

使用人员只负责投放材料，不负责整理。即使材料来自对话窗口、聊天记录、混合文件或多个项目混在一起，也要由系统自动分拣。

优先识别：

- 是否是剧本。
- 是否是分镜。
- 是否是剧本和分镜混合。
- 是否像最终稿。
- 是否像被废弃的草稿。
- 是否多个文件属于同一项目。
- 是否存在剧本和分镜配对关系。

不要因为文件混乱就停止处理。先输出待确认清单。

不要要求使用人员改文件名、建目录、写 notes、判断剧本和分镜关系或提炼规则。

如果系统无法根据文件名、集数、正文人物、地点和剧情判断关联关系，可以向使用人员提出简短确认问题。

## 未知材料二次判断规则

如果文件名扫描报告中出现“未知材料”，不要立刻要求使用人员解释。先读取正文做二次判断，并生成 `_reports/unknown-material-review-日期.md`。

二次判断至少写清：

- 正文类型：小说原文、剧本、分镜、剧目设定、人物小传、混合材料。
- 归属剧目：优先按文件名、正文标题、人物名和剧情设定判断。
- 建议关系：小说到剧本、剧本到分镜、总材料到多集分镜、观察样例或待确认。
- 可学习价值：能支持哪些题材规则、格式规则或评测任务。
- 暂不学习内容：哪些判断还缺正文对齐或用户确认。

只有在正文二次判断后仍无法确认关键对应关系时，才向使用人员提出简短问题。

## 文件关联关系规则

文件之间可能存在以下关系：

- `小说 -> 剧本`
- `剧本 -> 分镜`
- `小说或总剧本 -> 多集分镜`
- `多集剧本 -> 已收到的若干集分镜`
- `草稿 -> 定稿`
- `同一故事不同集`
- `同一故事不同版本`

判断依据：

- 文件名共享关键词。
- 文件名包含相同集数。
- 一个文件名含“剧本”，另一个文件名含“分镜”。
- 正文中的人物、地点、剧情一致。
- 用户明确说明二者有关联。

## 文件名优先成组规则

处理 `samples/` 时，先用文件名做第一轮关联。

步骤：

1. 去掉材料类型词：剧本、分镜、小说、改写稿、完本、草稿、定稿。
2. 保留标题词和集数词。
3. 标题词相同或高度重合的文件，先归入同一故事组。
4. 集数词相同的剧本和分镜，优先判断为同一集配对。
5. 如果只有故事组相同但集数不明确，标为“同故事组，待确认集数”。
6. 不要因为还没读取正文，就把文件名明显关联的文件拆开学习。

当前样例中的文件名关联：

- 新房：`新房剧本.docx` -> `新房分镜.docx`。
- 第十七集：`第十七集 - 剧本.doc` -> `第十七集 - 分镜.doc`。
- 八零杂货铺：`八零杂货铺.docx`、`八零杂货铺_AI漫剧前10集剧本改写稿.md`、`八零分镜第二集.docx`、`八零分镜第3集分镜.docx`。
- 欧皇盲盒：欧皇小说与第一集、第二集、第三集分镜属于同一系列。
- 少奶奶 / 司爷：`救命！司爷他快镇不住少奶奶了.txt` 与 `少奶奶分镜.doc` 属于同一故事组。

高价值配对：

- 剧本和分镜配对。
- 原始剧本和最终剧本配对。
- 原始分镜和最终分镜配对。

这类配对应优先学习。

## 一对多关系规则

不要默认样例都是一对一。AI 漫剧项目里，经常是一个小说或一个多集剧本，对应多份分集分镜。

判断方式：

- 文件名共享故事标题，例如“八零杂货铺”。
- 一个文件包含多集剧本或完整原文。
- 多个分镜文件各自带有集数词，例如“第二集”“第3集”。

处理方式：

- 先按故事建立总关联组，也就是一部剧一个目录。
- 再按集数建立 `episodes/` 分集样例目录。
- 当前只有第 2 集、第 3 集分镜，也照常学习这两集。
- 不把未收到的其他集分镜视为错误。
- 在报告中说明“当前只收到哪些集分镜”。

推荐结构：

```text
samples/剧目/<编号>-<剧名>/
├── 剧目说明.md
└── episodes/
    ├── 002-第二集/
    └── 003-第三集/
```

## 部分对应处理规则

文件名和集数能够配对，不代表正文一定完整覆盖。读取正文后如果发现分镜省略剧本开头、删掉尾钩、合并场次、重排场次或加入明显改写段，应标为“部分对应”。

部分对应样例仍然可以学习，但只能学习可靠片段：

- 已覆盖场次的镜头拆分。
- 经营、打脸、反转、情绪等明确镜头链。
- 分镜字段、时长、景别、运镜和音效习惯。
- 物品、界面、环境等画面化方法。

不能学习：

- 把缺失场次当作可删除规则。
- 把分镜未覆盖的尾钩当作不需要尾钩。
- 把改写痕迹当作原剧本必然产物。
- 把部分对应样例放入完整强基准。

`notes.md` 必须写清楚：

- 哪些场次已覆盖。
- 哪些场次缺失。
- 哪些地方可能被改写。
- 当前能学什么、暂不学什么。

## 不能直接提炼为正式规则的情况

- 只有一个样例支持。
- 样例质量未知。
- 用户没有确认满意。
- 样例之间互相冲突。
- 规则只适用于某个特殊项目。

## 低置信材料处理

以下材料只能进入观察，不得进入候选规则：

- 不知道是否最终稿。
- 无法判断项目归属。
- 明显是中间草稿。
- 与其他样例冲突，但缺少用户说明。
- 剧本和分镜只部分对应，且缺口未说明。

## 候选规则状态

- `待观察`：只有少量证据。
- `可讨论`：有多个样例支持，但还需用户确认。
- `建议采用`：有稳定证据，且不与现有规则冲突。
- `不采用`：来自失败样例或与项目目标冲突。

## 新样例进入 M4 的触发条件

新样例满足以下任一条件时，应建议进入 M4 评测：

- 属于 `samples/待补样例清单.md` 中的缺口题材。
- 有剧本和分镜完整强对应。
- 能补足某个候选规则的第二个或第三个独立证据。
- 用户明确说这是认可样例或最终稿。
- 与已确认规则冲突，需要验证规则边界。

进入 M4 前必须先判断样例关系：

- 完整强对应：可做评测基准。
- 部分对应：只做缺口识别或片段学习评测。
- 观察样例：可做链路试跑，不直接做强映射评测。
- 待确认：先提问确认，不进入规则升级。
