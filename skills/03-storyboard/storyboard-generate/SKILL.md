# 分镜生成

## 作用

把剧本转成 AI 漫剧分镜表。

这个技能可以处理两种情况：

- 剧本已经通过评审，直接生成分镜。
- 剧本存在小缺口，但不影响分镜，可以边生成边标注风险。

如果剧本结构混乱，应先交给 `script-review-rewrite`。

默认目标是 AI 漫剧。分镜要服务后续 AI 画面生成、AI 视频生成、字幕、音效和剪辑节拍，不默认按传统影视现场拍摄来写。

## 什么时候使用

遇到以下情况时使用：

- 用户说“根据这个剧本生成分镜”。
- 用户提供已认可剧本，希望直接拆镜头。
- 剧本评审后已经形成定稿。
- 用户需要检查分镜是否可拍、是否覆盖剧本。

## 分镜生成流程

1. 识别剧本场次和节拍。
2. 为每个节拍确定镜头目的。
3. 设计景别、运镜、构图视角、画面主体和角色动作。
4. 对应对白或旁白。
5. 标注时长估计和连续性备注。
6. 输出分镜表。
7. 自查是否漏掉关键剧情、重复镜头、运镜混乱、构图单一或画面空泛。

## 输出格式

默认使用逐镜头文本块，不使用表格。

```text

地点：无
时间：无

```

## 质量门槛

- 分镜必须是画面，不是剧本复述。
- 每个镜头都要有明确叙事目的。
- 镜头要覆盖关键剧情，不要漏掉转折。
- 景别和镜头运动要服务情绪，不要为了花哨而花哨。
- 运镜要合理变化，不要全是固定镜头，也不要乱运镜。
- 人物对话不要频繁正面怼脸，不要对同一角色台词硬切多角度多景别。
- 构图视角要根据剧情变化，可使用正面、正四三、反打、前景遮挡、平视、仰视、俯视、低视角等。
- 需要切镜时，优先考虑其他角色反应、身体细节反应或相关物品画面；没有合适反应不要硬给。
- 对白和旁白要能对应到镜头。
- 画面主体要清楚，便于后续转成 AI 画面或视频提示词。
- 避免单镜头塞入过多人物、动作和复杂空间关系。
- 发现剧本缺口时要标注，不要悄悄补剧情。
- 输出格式必须遵守项目分镜标准格式：场次、地点、时间、人物、镜号、景别、运镜、情绪/动作、音效、台词、时长。

## 分镜评审

生成后自查：

- 是否有空泛画面，如“展示冲突”“表现悲伤”但没有具体画面。
- 是否连续多个镜头景别完全重复。
- 是否连续大量固定镜头或乱用运镜。
- 是否连续大量正面怼脸对白。
- 是否一场戏全是平视视角，缺少合理构图变化。
- 是否对同一角色台词做了无意义多角度切镜。
- 是否动作不可拍。
- 是否角色位置前后矛盾。
- 是否缺少转场。
- 是否有对白但没有对应人物或画面。

## 分镜不通过后的去向

分镜评审必须输出后续流程，不允许只写“不通过”。

### 1. 通过

条件：

- 分镜覆盖剧本关键内容。
- 镜头顺序清楚。
- 字段完整。
- 画面具体。
- 可进入 `storyboard.final.md`。

后续：

```text
进入分镜定稿
```

### 2. 分镜小修

适用问题：

- 个别镜头景别不合适。
- 运镜缺失或不自然。
- 音效、台词、时长字段漏填。
- 情绪/动作描述太空。
- 少量转场不清。

后续：

```text
修订对应镜头，不返回剧本
```

### 3. 重新生成分镜

适用问题：

- 漏掉关键剧情。
- 镜头顺序整体不合理。
- 某一场拆镜节奏错误。
- 镜头重复太多。
- 分镜密度和目标平台不匹配。

后续：

```text
保留剧本，重新生成对应场次或整段分镜
```

### 4. 返回剧本评审 / 改写

适用问题：

- 剧本缺少可拍动作。
- 冲突太抽象，无法画面化。
- 人物动机不清，镜头无法成立。
- 台词承担过多信息，但没有对应动作。
- 场景跳跃导致分镜无法连续。

后续：

```text
停止硬拆分镜，返回 script-review-rewrite
```
