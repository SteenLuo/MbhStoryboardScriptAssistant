# 剧本漫剧适配与立项分析

## 作用

评估已有剧本是否适合做成 AI 漫剧，并给出甲方可读的立项建议、适配风险和改编策略。

本技能是独立环节，和 `script-hard-issue-review` 分开：

- `script-hard-issue-review` 判断剧本内部有没有硬伤、逻辑漏洞、伏笔未兑现和人物撕裂。
- 本技能判断剧本作为 AI 漫剧项目是否值得推进、适不适合做、怎么改更适合做。

如果已有硬伤评审报告，可以引用严重硬伤作为适配风险；不要重复输出全量硬伤清单。若没有硬伤报告，只指出影响漫剧适配的关键阻断点，不替代硬伤评审。

## 评估边界

- 默认按竖屏、短集、AIGC 漫剧流水线评估。
- 默认不输出具体成本金额或报价区间。
- 默认不承诺抓取各平台实时 Top100。
- 不把经验判断写成市场数据事实。
- 不把题材热度等同于项目可做性。
- 不做完整分镜拆解，但要判断分镜友好度和视觉化难度。
- 不直接整篇改写，除非用户明确要求进入改写。
- 不评小说原文，除非用户明确要求做 IP 源文本到剧本的适配对照。

## 工作流程

1. 确认材料范围：剧本文件、集数、单集时长、目标平台、目标受众、是否有硬伤报告、是否有市场/竞品资料。
2. 标注评估模式：无数据模式、资料增强模式或联网检索模式。
3. 建立分析台账：
   - **题材台账**：题材、年代、核心类型、主情绪、受众假设。
   - **人设台账**：主角、反派、CP/亲情/对抗关系、身份反差、情绪债。
   - **钩子台账**：前 3 集/前 5 集开场、冲突、结尾钩子；全剧单集钩子矩阵。
   - **爽点台账**：爽点类型、出现频率、兑现周期、情绪释放节点。
   - **视觉台账**：可画场面、动作、表情、道具、空间调度、旁白和心理活动比例。
   - **资产台账**：核心角色、核心场景、服装造型、年代道具、群众戏、动作戏、复用可能性。
   - **风险台账**：合规、授权、平台表达、制作复杂度、潜在流失原因。
4. 按固定报告结构输出 8 个大章。
5. 用等级制评级，不用百分制。
6. 每个核心结论必须写清“结论、证据、影响、建议”。
7. 输出当前版本判断和改编后潜力判断。
8. 输出强制改编策略清单和最终自检。

## 市场数据模式

| 模式 | 触发条件 | 输出边界 |
| --- | --- | --- |
| 无数据模式 | 只有剧本文本 | 只做题材、受众、卖点、留存、差异化和漫剧适配的文本内推断。 |
| 资料增强模式 | 用户提供榜单、截图、Excel、报告、竞品资料 | 基于用户资料归纳题材趋势、竞品特征、平台偏好和差异化风险。 |
| 联网检索模式 | 用户明确要求查公开资料、最新趋势、平台榜单或竞品 | 只查公开可访问页面和公开报告；提供来源链接；不绕登录、不破反爬、不承诺全平台 Top100。 |

### 证据等级

| 等级 | 含义 |
| --- | --- |
| A | 用户提供的明确数据、榜单、后台截图、授权报告。 |
| B | 公开行业报告、公开榜单、可访问网页。 |
| C | 基于剧本文本和行业经验的推断。 |
| D | 数据不足，只能提出待验证假设。 |

市场判断必须标注证据等级。证据不足时，使用“文本内推断”“待验证假设”，不要写成确定事实。

## 评级体系

使用“高 / 较高 / 中 / 较低 / 低”或“低 / 中 / 高 / 极高”评级，并给出触发依据。

| 评级项 | 评级口径 | 必须说明的依据 |
| --- | --- | --- |
| 市场与受众适配 | 高 / 较高 / 中 / 较低 / 低 | 题材吸引力、受众明确度、卖点差异化、传播转化点。 |
| 前几集留存能力 | 高 / 较高 / 中 / 较低 / 低 | 开场目标、冲突出现时间、主角困境、前三秒/前三集钩子、集尾悬念。 |
| 爽点与追更驱动力 | 高 / 较高 / 中 / 较低 / 低 | 爽点类型、出现频率、兑现周期、反转密度、连续期待。 |
| 漫剧视觉化适配度 | 高 / 较高 / 中 / 较低 / 低 | 是否能转成画面、是否依赖旁白、是否有动作/表情/空间/道具抓手。 |
| AIGC 制作可控性 | 高 / 较高 / 中 / 较低 / 低 | 角色一致性、服装道具、场景复用、群众戏、动作戏、表情控制。 |
| 制作复杂度 | 低 / 中 / 高 / 极高 | 角色数量、场景数量、年代/特殊题材、资产复用率、镜头动态化难度。 |
| 工期风险 | 低 / 中 / 高 / 极高 | 只说明风险来源，不给天数；例如角色多、场景散、年代道具多、群众戏多、返修概率高。 |
| 合规与平台风险 | 低 / 中 / 高 / 极高 | 暴力、违法、婚恋伦理、未成年人、封建迷信、歧视表达、授权和平台表达边界。 |

## 固定报告结构

正式报告使用 8 个大章。章节可以按项目情况增减小节，但不要漏掉核心判断。

### 1. 评估前提与证据模式

- 项目名称、剧本集数、材料来源、是否有硬伤报告。
- 默认目标形态：竖屏短集 AI 漫剧；若用户指定平台、时长、目标集数或质量档位，按用户参数。
- 市场数据模式：无数据 / 资料增强 / 联网检索。
- 证据等级说明。
- 缺失参数：目标平台、目标受众、单集时长、目标集数、竞品资料、生产标准等。

### 2. 总体评级与立项建议

固定输出：

| 结论项 | 输出 |
| --- | --- |
| 当前版本适配度 | 高 / 较高 / 中 / 较低 / 低 |
| 改编后潜力 | 高 / 较高 / 中 / 较低 / 低 |
| 立项建议 | 建议推进 / 谨慎推进 / 暂缓推进 / 不建议推进 / 先试制验证 |
| 改编工作量 | 轻改 / 中改 / 重构 |
| 重大暂缓因素 | 无 / 有，列出原因 |
| 数据可信度 | 高 / 中 / 低 |

“一票否决”只用于违法违规、授权缺失、明显不适配目标平台等根本性问题。其他情况称为“重大暂缓因素”。

### 3. 市场与受众适配

不要只写“市场潜力高/低”。拆成：

- 题材吸引力。
- 受众明确度。
- 卖点差异化。
- 竞品替代性。
- 文本内差异化判断。

没有用户资料或公开数据时，只能写“文本内推断”。有榜单、竞品或公开资料时，再输出竞品/市场资料分析。

### 4. 卖点、人设与传播转化

重点判断剧本能否被标题、封面、前三秒、短视频切片卖出去。

必须覆盖：

- 一句话卖点是否清楚。
- 主角商业力：身份反差、目标、困境、记忆点、成长或逆袭空间。
- 反派/阻力商业力：压迫感、可恨点、对抗强度。
- CP/亲情/对抗关系：情绪债、关系拉扯、可传播桥段。
- 标题、简介、封面冲突、爆点切片的可提取性。
- 观众潜在拒看理由：卖点不清、主角弱、爽点慢、题材旧、差异化不足等。

### 5. 留存、爽点与追更结构

不要只看前三集，也要扫全剧中段掉线风险。

必须输出：

| 集数/段落 | 开场钩子 | 本集核心冲突 | 爽点/情绪点 | 结尾钩子 | 追更风险 |
| --- | --- | --- | --- | --- | --- |

检查：

- 前 3 集/前 5 集是否快速建立目标、困境、冲突和主角期待。
- 单集是否有明确事件，不只是解释和过渡。
- 爽点类型、出现频率、兑现周期是否稳定。
- 情绪曲线是否有压迫、期待、释放、反转。
- 中后段是否重复冲突、拖延兑现或突然换题材。
- 是否具备付费/转化卡点、集尾反转和连续期待。

### 6. 漫剧视觉化与 AIGC 制作可控性

必须把“适合漫剧”落到可制作因素上：

- 分镜友好度：台词、旁白、心理活动能否转成画面。
- 画面抓手：动作、表情、道具、空间、光影、冲突场面是否足够。
- 角色资产：角色数量、造型变化、辨识度、一致性维护难度。
- 场景资产：核心场景数量、年代场景、室内外比例、复用率。
- 高难内容：群众戏、大场面、打斗、追逐、特殊道具、复杂服化道。
- 返修压力：哪些内容会导致 AIGC 出图/视频不稳定。

不输出金额成本。只输出制作复杂度、工期风险、资源压力和降复杂度建议。

### 7. 风险清单

固定分为：

| 风险类型 | 等级 | 证据/依据 | 影响 | 建议 |
| --- | --- | --- | --- | --- |
| 合规风险 | 低/中/高/极高 |  |  |  |
| 授权/权利风险 | 低/中/高/极高 |  |  |  |
| 平台表达风险 | 低/中/高/极高 |  |  |  |
| 制作复杂度风险 | 低/中/高/极高 |  |  |  |
| 潜在流失原因 | 低/中/高/极高 |  |  |  |

权利与素材风险包括小说/IP 授权、真人事件、品牌、歌曲、历史人物、肖像或平台素材使用等。材料不足时标注“需甲方确认”。

### 8. 改编策略清单与执行优先级

综合型报告必须输出改编策略清单，不能只给评级。

| 优先级 | 改编方向 | 对应问题 | 涉及集数/段落 | 预期收益 | 风险 | 验收标准 |
| --- | --- | --- | --- | --- | --- | --- |

改编策略重点解决：

- 强化开场钩子。
- 明确一句话卖点。
- 提升主角、反派和关系线商业力。
- 提高爽点密度和兑现节奏。
- 增强集尾悬念和追更动力。
- 将旁白、解释和心理活动转成画面动作。
- 合并低价值角色和场景。
- 提高资产复用率。
- 降低群众戏、动作戏、年代道具和复杂场景压力。
- 降低合规、授权和平台表达风险。

## 试制验证建议

证据不足或结论不确定时，不要强行给确定性市场结论。建议输出试制验证路径：

| 验证项 | 做法 | 验证目的 |
| --- | --- | --- |
| 前 1-3 集试制 | 制作样片或分镜样段 | 验证人设、钩子、画面风格是否成立。 |
| 标题/封面测试 | 准备 2-3 组卖点包装 | 验证观众是否一眼理解。 |
| 爆点片段测试 | 提取强冲突桥段 | 验证传播和投流素材潜力。 |
| 角色形象测试 | 生成主角、反派、关键关系人设图 | 验证记忆点和一致性。 |
| AIGC 样片测试 | 测试关键场景、年代道具、动作镜头 | 验证制作可控性和返修压力。 |

## 结论证据链

每个核心结论都按以下格式写：

```markdown
- 结论：
- 证据：
- 影响：
- 建议：
```

禁止只写“适配度较高”“市场潜力一般”“制作难度中等”而不给依据。

## 常见误区

- 用热门题材替代项目判断。
- 把个人经验写成平台数据。
- 没有竞品资料却输出确定性竞品结论。
- 只看前三集，不看全剧中后段掉线风险。
- 只说“画面感强”，不分析分镜友好度和 AIGC 可控性。
- 只给“建议推进”，不给当前版和改编后潜力的区别。
- 把硬伤评审重新做一遍，导致报告和 `script-hard-issue-review` 重复。
- 给出具体成本金额，但没有企业内部单价、人效、工具链和质量档位。

## 最终自检

输出正式报告前必须确认：

- 已明确评估模式和证据等级。
- 已输出等级制评级，而不是百分制伪精确评分。
- 已输出当前版本判断和改编后潜力判断。
- 已列出重大暂缓因素或说明没有发现。
- 已区分市场事实、公开资料和文本内推断。
- 已检查前 3 集/前 5 集留存，并扫全剧单集钩子矩阵。
- 已分析卖点、人设商业力和传播素材可提取性。
- 已分析视觉化、分镜友好度和 AIGC 制作可控性。
- 已输出制作复杂度和工期风险来源，但没有输出具体金额成本。
- 已输出潜在流失原因和试制验证建议。
- 已输出可执行改编策略清单，并包含优先级、涉及位置、预期收益和验收标准。
- 报告没有替代 `script-hard-issue-review` 做全量硬伤清单。

## 与其他项目技能的关系

- 需要完整剧本硬伤清单时，使用 `script-hard-issue-review`。
- 需要按评审结论改写剧本时，使用 `script-review-rewrite`。
- 需要判断剧本是否适合 AI 漫剧立项、市场与受众是否匹配、AIGC 制作是否可控、应如何改编时，使用本技能。
- 需要生成分镜时，交给 `storyboard-generate`。
