# 小说输入整理

## 作用

把用户提供的小说、章节片段或已有剧本整理成后续 AI 漫剧技能可用的基础信息、小说总纲、分集大纲和剧情分析。

这个技能不负责正式写剧本，也不负责生成分镜。它只负责判断“手里的材料是什么、应该走哪条流程、缺什么信息”。

## 什么时候使用

遇到以下情况时使用：

- 用户粘贴小说原文，希望生成剧本和分镜。
- 用户提供章节片段，但没有明确类型和目标平台。
- 用户提供已有剧本，不确定是要评审、改写，还是直接生成分镜。
- 用户同时提供小说、剧本、分镜片段，需要整理材料关系。

默认目标是 AI 漫剧。除非用户明确指定其他用途，否则输入整理时应按 AI 漫剧改编来判断目标平台、节奏、画面化程度和后续技能链路。
- 新建一次 `runs/` 运行目录前，需要形成基础信息。

## 三种入口判断

### 入口一：小说到剧本再到分镜

判断条件：

- 用户主要提供小说原文。
- 没有已有剧本，或已有剧本只是草稿。
- 用户希望先生成或调整剧本。

后续技能：

```text
novel-intake -> script-review-rewrite -> storyboard-generate
```

### 入口二：已有剧本评审改写后生成分镜

判断条件：

- 用户主要提供剧本。
- 用户希望优化剧本、检查结构、改写节奏或对白。
- 分镜应该基于改写后的剧本生成。

后续技能：

```text
novel-intake -> script-review-rewrite -> storyboard-generate
```

### 入口三：剧本直接生成分镜

判断条件：

- 用户提供已认可剧本。
- 用户明确说“不用改剧本”“直接分镜”“按这个剧本出分镜”。
- 剧本结构足够清楚，可以直接拆镜头。

后续技能：

```text
novel-intake -> storyboard-generate
```

## 工作流程

1. 识别材料类型：小说、剧本、分镜、混合材料。
2. 提取基础信息：标题、类型、题材、人物、关系、冲突、时间地点。
3. 做正向分析：形成小说总纲、每集大纲、每集 2 到 4 个爽点或剧情高潮点。
4. 做负面分析：检查世界观、人物行为、因果、时空、人物状态、常识、对话信息。
5. 判断目标：生成剧本、评审改写剧本、直接生成分镜。
6. 判断缺口：是否缺平台、时长、风格、目标受众、已有样例。
7. 输出结构化整理结果。

## 质量门槛

- 不要直接改写用户故事。
- 不要擅自补完整个世界观。
- 不要把剧本任务误判成小说任务。
- 如果可以直接进入分镜，要说明为什么剧本已经足够稳定。
- 如果不适合直接进入分镜，要指出最小修改点。
- 每集爽点或高潮点少于 2 个时，要标记为“高潮不足”。
- 每集爽点或高潮点超过 4 个时，要提示“节奏可能过密”。
- 必须检查单集剧情和整体剧情，不只看当前片段。
- 负面分析必须覆盖世界观、人物行为、因果、时空、人物状态、常识、对话信息七类问题。
